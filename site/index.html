Protocol specification

An open protocol for addressable LED controllers

Opalinx defines implementation-independent messages for communication between host software and addressable LED controllers over reliable byte streams.

Read the specification Open Workbench

Protocol scope

The core specifies framing, device information, configuration, pixel transfer, coordinated output, and recovery without prescribing a software or firmware architecture.

01

Framing and validation

COBS framing, CRC protection, defined errors, and transaction correlation over reliable byte streams.

02

LED data and output

RGB and RGBW pixel data, channel configuration, and coordinated multi-channel Show operations.

03

Extension points

Information records and namespaced vendor messages allow additive extensions to the core protocol.

Reference tool

Opalinx Workbench

The Workbench connects to compatible USB controllers through Web Serial for device inspection, configuration, and protocol testing. It requires a browser with Web Serial support.
